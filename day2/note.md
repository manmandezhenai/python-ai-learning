## 一、分支结构（条件判断）

分支结构的核心是「根据不同的条件，执行不同的代码逻辑」，是程序实现决策的基础。

### 1. 单分支结构（if）

- 语法：当条件表达式结果为 `True` 时，执行缩进内的代码块；为 `False` 则跳过

```python
if 条件表达式:
    条件成立时执行的代码
```

### 2. 双分支结构（if-else）

- 语法：条件成立执行 `if` 代码块，不成立则执行 `else` 代码块，二选一

```python
if 条件表达式:
    条件成立时执行的代码
else:
    条件不成立时执行的代码
```

### 3. 多分支结构（if-elif-else）

- 语法：用于 3 个及以上的条件判断，从上到下依次匹配，**匹配到第一个成立的条件后，后续分支不再执行**
- `elif` 可以有多个，`else` 是可选的，放在所有分支最后，作为兜底

```python
if 条件表达式1:
    条件1成立时执行的代码
elif 条件表达式2:
    条件2成立时执行的代码
elif 条件表达式3:
    条件3成立时执行的代码
else:
    所有条件都不成立时执行的代码
```

### 4. 嵌套分支结构

- 语法：在一个分支内部，再嵌套一层 / 多层分支，用于复杂条件判断
- 注意：嵌套层级不宜过多，否则会降低代码可读性，建议不超过 3 层

### 5. 三元表达式（条件表达式）

- 作用：用一行代码简化双分支结构，让代码更简洁
- 语法：`结果1 if 条件表达式 else 结果2`
    - 条件成立，返回 `结果1`；条件不成立，返回 `结果2`

## 二、循环结构

循环结构的核心是「让一段代码重复执行多次」，避免冗余代码。

### 1. while 循环

- 核心：「先判断条件，再执行循环体」，只要条件表达式为 `True`，就会重复执行循环体代码

```python
while 条件表达式:
    循环体代码
    循环变量更新
```

### 2. for 循环

- 核心：「遍历可迭代对象，依次取出每个元素执行循环体」，是 Python 中最常用的循环
- 可迭代对象：字符串、列表、元组、字典、集合、range() 对象等

```python
for 临时变量 in 可迭代对象:
    循环体代码
```

#### 重点：range() 函数

- 作用：生成一个整数序列，是 for 循环最常用的搭档，专门用于控制循环次数
- 语法：`range(start, end, step)`
    - start：起始值（默认为 `0`）
    - end：结束值（默认为 `end - 1`）
    - step：步长（默认为 `1`）

### 3. 循环控制关键字

| 关键字        | 作用                                |
|------------|-----------------------------------|
| `break`    | 立即终止当前整个循环                        |
| `continue` | 跳过本次循环剩余的代码，直接进入下一次循环             |
| `else`     | 只有循环正常结束（没有被 break 终止）时，else 才会执行 |

### 4. while 与 for 循环的选型

| 循环类型     | 核心适用场景                | 示例                      |
|----------|-----------------------|-------------------------|
| while 循环 | 循环次数不确定，只知道循环终止条件     | 猜数字游戏、用户输入校验、持续监听       |
| for 循环   | 循环次数确定，遍历可迭代对象、批量数据处理 | 遍历列表 / 字符串、固定次数循环、数据预处理 |

## 三、循环嵌套

- 定义：在一个循环体内部，再嵌套一层 / 多层循环，外层循环执行一次，内层循环执行完整的一轮
- 核心应用场景：九九乘法表、矩阵运算、多维数据遍历

```python
for 外层变量 in 外层可迭代对象:
    外层代码
    for 内层变量 in 内层可迭代对象:
        内层代码
```

## 四、列表推导式

- 作用：用一行代码快速生成列表，替代 for 循环 + append () 的冗余写法，代码更简洁、执行效率更高
- 是 Python 中最具特色的语法之一，AI 数据预处理、批量数据生成中高频使用

### 1. 基础列表推导式

- 语法：`[表达式 for 临时变量 in 可迭代对象]`
- 等价于：先执行 `for` 循环，把每次循环的表达式结果添加到新列表中

### 2. 带条件筛选的列表推导式

- 语法：`[表达式 for 临时变量 in 可迭代对象 if 筛选条件]`
- 作用：只保留满足 `if` 条件的元素，执行表达式后加入列表

### 3. 带 if-else 双条件的列表推导式

- 语法：`[结果1 if 条件 else 结果2 for 临时变量 in 可迭代对象]`
- 注意：双条件时，`if-else` 要放在 `for` 循环前面

### 4. 嵌套循环的列表推导式

- 语法：`[表达式 for 外层变量 in 外层可迭代对象 for 内层变量 in 内层可迭代对象]`
- 循环顺序：外层循环在前，内层循环在后，和嵌套 `for` 循环的顺序一致

## 五、今日易错点总结

1. **缩进错误**：分支 / 循环体的代码没有正确缩进，或混用 Tab 和空格，导致语法错误 / 逻辑错误
2. **漏写冒号**：if / elif / else / for / while 语句末尾的英文冒号:漏写，直接报错
3. **混淆赋值与判断**：把条件判断的==写成赋值的=，导致语法错误
4. **死循环**：while 循环忘记更新循环变量，导致条件永远为 True，程序卡死
5. **break/continue 作用范围混淆**：嵌套循环中，误以为 break 能跳出所有层循环，实际只能跳出当前层
6. **循环 else 子句理解错误**：误以为循环结束就会执行 else，实际只有未被 break 终止的正常结束才会执行
7. **列表推导式条件位置写错**：单条件筛选的 if 写在了 for 前面，双条件的 if-else 写在了 for 后面，导致语法错误
8. **range () 左闭右开规则记错**：循环次数少一次，比如想循环 10 次，写了 range (1,10)，实际只有 9 次
