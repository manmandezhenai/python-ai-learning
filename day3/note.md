## 一、四大核心容器深入讲解

### 1. 列表（list）：最常用的可变有序容器

- 底层特性：动态数组，支持随机访问，增删改查灵活，内存连续分配
- 适用场景：存储有序、需要频繁修改的数据，如数据集列表、特征向量、日志记录等

#### 1.1 列表常用方法（按使用频率排序）

| 方法                                    | 作用                    | 示例                                  | 返回值    |
|---------------------------------------|-----------------------|-------------------------------------|--------|
| `append(item)`                        | 在列表末尾添加一个元素           | `[1,2].append(3)` → `[1,2,3]`       | None   |
| `extend(iterable)`                    | 在列表末尾添加可迭代对象的所有元素     | `[1,2].extend([3,4])` → `[1,2,3,4]` | None   |
| `insert(index, item)`                 | 在指定索引位置插入元素           | `[1,3].insert(1,2)` → `[1,2,3]`     | None   |
| `pop(index=-1)`                       | 删除并返回指定索引的元素，默认删除最后一个 | `[1,2,3].pop()` → 3，列表变为`[1,2]`     | 被删除的元素 |
| `remove(item)`                        | 删除第一个出现的指定元素，元素不存在会报错 | `[1,2,1].remove(1)` → `[2,1]`       | None   |
| `clear()`                             | 清空列表所有元素              | `[1,2].clear()` → `[]`              | None   |
| `index(item, start=0, end=len(list))` | 返回第一个匹配元素的索引，不存在报错    | `[1,2,3].index(2)` → 1              | 索引值    |
| `count(item)`                         | 统计元素在列表中出现的次数         | `[1,2,1].count(1)` → 2              | 次数     |
| `sort(key=None, reverse=False)`       | 原地排序，默认升序             | `[3,1,2].sort()` → `[1,2,3]`        | None   |
| `reverse()`                           | 原地反转列表                | `[1,2,3].reverse()` → `[3,2,1]`     | None   |
| `copy()`                              | 浅拷贝列表                 | `a=[1,2], b=a.copy()` → `b=[1,2]`   | 新列表    |

#### 1.2 列表易错点

- `append()`和`extend()`的区别：`append()`添加整个对象，`extend()`添加对象的元素
- `pop()` 和 `remove()` 的区别：`pop()` 按索引删除，返回被删元素；`remove()` 按值删除，无返回值
- `sort()` 是原地排序，会修改原列表；如果不想修改原列表，用 `sorted()` 函数（返回新列表）

### 2. 字典（dict）：最常用的键值对容器

- 底层特性：哈希表，键必须是不可变类型（int、str、tuple 等），键唯一，查询速度极快（O (1) 时间复杂度）
- 适用场景：存储结构化数据、键值映射关系，如学生信息、配置参数、API 响应数据等（AI 开发中最常用的容器）

#### 2.1 字典常用方法（按使用频率排序）

| 方法                       | 作用                              | 示例                                                      | 返回值        |
|--------------------------|---------------------------------|---------------------------------------------------------|------------|
| `get(key, default=None)` | 获取指定键的值，键不存在返回 default，不会报错     | `{"a":1}.get("b",0) → 0`                                | 值或 default |
| `keys()`                 | 返回所有键的视图对象                      | `{"a":1,"b":2}.keys() → dict_keys(['a','b'])`           | 键视图        |
| `values()`               | 返回所有值的视图对象                      | `{"a":1,"b":2}.values() → dict_values([1,2])`           | 值视图        |
| `items()`                | 返回所有键值对的视图对象                    | `{"a":1,"b":2}.items() → dict_items([('a',1),('b',2)])` | 键值对视图      |
| `update(dict2)`          | 用 dict2 的键值对更新原字典，键存在则覆盖，不存在则添加 | `{"a":1}.update({"b":2,"a":3}) → {"a":3,"b":2}`         | None       |
| `pop(key, default=None)` | 删除并返回指定键的值，键不存在返回 default       | `{"a":1}.pop("a") → 1`                                  | 	被删除的值     |
| `popitem()`              | 删除并返回最后一个插入的键值对（Python3.7+）     | `{"a":1,"b":2}.popitem() → ('b',2)`                     | 键值对元组      |
| `clear()`                | 清空字典所有键值对                       | `{"a":1}.clear() → {}`                                  | None       |

#### 2.2 字典易错点

- 键必须是不可变类型：列表、字典不能作为键，否则会报 `TypeError`
- 直接用 `dict[key]` 访问不存在的键会报错，推荐用 `get()` 方法，避免程序崩溃
- 视图对象（keys/values/items）会随原字典动态变化，如需固定值，转成列表：`list(dict.keys())`

### 3. 集合（set）：无序不可重复的容器

- 底层特性：基于哈希表实现，元素唯一，无序，不支持索引访问
- 适用场景：去重、集合运算（交集、并集、差集），如数据去重、用户标签对比等

#### 3.1 集合常用方法（按使用频率排序）

| 方法                   | 作用                | 示例                                 | 返回值    |
|----------------------|-------------------|------------------------------------|--------|
| `add(item)`          | 添加一个元素            | `{1,2}.add(3) → {1,2,3}`           | None   |
| `remove(item)`       | 删除指定元素，元素不存在会报错   | `{1,2}.remove(1) → {2}`            | None   |
| `discard(item)`      | 删除指定元素，元素不存在不会报错  | `{1,2}.discard(3) → 无变化`           | None   |
| `pop()`              | 随机删除并返回一个元素       | `{1,2,3}.pop() → 随机一个元素`           | 被删除的元素 |
| `clear()`            | 清空集合所有元素          | `{1,2}.clear() → set()`            | None   |
| `union(set2)`        | 求并集               | `{1,2}.union({2,3}) → {1,2,3}`     | 新集合    |
| `intersection(set2)` | 求交集               | `{1,2}.intersection({2,3}) → {2}`  | 新集合    |
| `difference(set2)`   | 求差集（原集合有，set2 没有） | `{1,2}.difference({2,3}) → {1}`    | 新集合    |
| `issubset(set2)`     | 判断是否是 set2 的子集    | `{1,2}.issubset({1,2,3}) → True`   | 布尔值    |
| `issuperset(set2)`   | 	判断是否包含 set2      | `{1,2,3}.issuperset({1,2}) → True` | 布尔值    |

#### 3.2 集合易错点

- 空集合必须用 `set()` 定义，`{}` 定义的是空字典
- 集合元素必须是不可变类型，列表、字典不能作为集合元素
- 集合是无序的，不能通过索引访问元素，也不能保证元素顺序

### 4. 元组（tuple）：不可变的有序容器

#### 4.1 元组常用方法

元组因为不可变，所以只有两个常用方法：

- `count(item)`：统计元素出现次数
- `index(item)`：返回第一个匹配元素的索引

#### 4.2 元组的特殊特性

- 单元素元组必须加逗号：`(1,)`，否则会被识别为普通变量
- 元组的不可变是指**元素的引用不可变**，如果元组包含可变元素（如列表），可变元素的内容可以修改

## 二、深拷贝与浅拷贝（面试高频考点，必须吃透）

### 1. 核心概念

- **赋值（=）**：只是复制对象的引用，两个变量指向同一个内存地址，修改一个会影响另一个
- **浅拷贝（copy.copy ()）**：创建一个新对象，但新对象中的元素仍然是原对象元素的引用
- **深拷贝（copy.deepcopy ()）**：创建一个完全独立的新对象，递归复制原对象的所有元素，包括嵌套的可变对象

### 2. 适用场景对比

| 拷贝类型 | 适用场景                  | 优点           | 缺点          |
|------|-----------------------|--------------|-------------|
| 赋值   | 不需要独立副本，只是给对象起别名      | 速度最快，内存占用最小  | 完全共享，修改互相影响 |
| 浅拷贝  | 原对象只有不可变元素，或只需要修改顶层元素 | 速度较快，内存占用较小	 | 嵌套可变元素仍然共享  |
| 深拷贝  | 原对象包含嵌套可变元素，需要完全独立的副本 | 完全独立，修改互不影响  | 速度最慢，内存占用最大 |

## 三、容器的通用操作

### 1. 容器遍历

#### 1.1 列表 / 元组遍历

```python
# 基础遍历
for item in [1, 2, 3]:
    print(item)

# 带索引遍历（推荐）
for index, item in enumerate([1, 2, 3]):
    print(f"索引{index}：{item}")
```

#### 1.2 字典遍历

```python
# 遍历键
for key in {"a": 1, "b": 2}.keys():
    print(key)

# 遍历值
for value in {"a": 1, "b": 2}.values():
    print(value)

# 遍历键值对（最常用）
for key, value in {"a": 1, "b": 2}.items():
    print(f"{key}: {value}")
```

#### 1.3 集合遍历

```python
for item in {1, 2, 3}:
    print(item)
```

### 2. 容器筛选

- 推荐使用**列表推导式**（Day2 学过），简洁高效
- 复杂筛选可以结合 `filter()` 函数

```python
# 列表推导式筛选
nums = [1, 2, 3, 4, 5, 6]
even_nums = [num for num in nums if num % 2 == 0]

# filter()函数筛选
even_nums = list(filter(lambda x: x % 2 == 0, nums))
```

### 3. 容器排序

#### 3.1 列表排序

- `list.sort()`：原地排序，修改原列表
- `sorted(list)`：返回新的排序后的列表，不修改原列表
- 支持 `key` 参数自定义排序规则，`reverse` 参数控制升序 / 降序

```python
# 基础排序
nums = [3, 1, 4, 2]
sorted_nums = sorted(nums)  # 升序：[1,2,3,4]
sorted_nums_desc = sorted(nums, reverse=True)  # 降序：[4,3,2,1]

# 自定义排序：按字符串长度排序
words = ["python", "ai", "java", "data"]
sorted_words = sorted(words, key=lambda x: len(x))
# 结果：['ai', 'java', 'data', 'python']
```

#### 3.2 字典排序

字典本身无序（Python3.7 + 保留插入顺序），排序后返回列表

```python
# 按键排序
d = {"b": 2, "a": 1, "c": 3}
sorted_by_key = sorted(d.items(), key=lambda x: x[0])
# 结果：[('a',1), ('b',2), ('c',3)]

# 按值排序
sorted_by_value = sorted(d.items(), key=lambda x: x[1], reverse=True)
# 结果：[('c',3), ('b',2), ('a',1)]
```

### 4. 容器合并

```python
# 列表合并
list1 = [1, 2]
list2 = [3, 4]
merged_list = list1 + list2  # 结果：[1,2,3,4]

# 字典合并（Python3.9+）
dict1 = {"a": 1, "b": 2}
dict2 = {"b": 3, "c": 4}
merged_dict = dict1 | dict2  # 结果：{"a":1, "b":3, "c":4}

# 集合合并
set1 = {1, 2}
set2 = {2, 3}
merged_set = set1 | set2  # 结果：{1,2,3}
```

## 四、今日易错点总结

1. **列表 append 与 extend 混淆**：`append` 添加整个对象，`extend` 添加对象的元素
2. **字典键类型错误**：使用列表、字典等可变类型作为字典键，导致 `TypeError`
3. **空集合定义错误**：用 `{}` 定义空集合，实际生成空字典
4. **单元素元组漏写逗号**：`(1)` 被识别为 `int`，`(1,)` 才是 `tuple`
5. **深拷贝浅拷贝混淆**：修改嵌套可变元素时，浅拷贝会影响原对象
6. **字典直接访问不存在的键**：用 `dict[key]` 访问不存在的键会报错，推荐用 `get()`
7. **sort 与 sorted 混淆**：`sort` 是原地排序，`sorted` 返回新列表
8. **集合元素类型错误**：集合元素必须是不可变类型，列表不能作为集合元素
